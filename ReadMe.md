Когда у вас много тестов, запуск всех сразу может быть медленным, особенно в больших проектах. Jest позволяет запускать только конкретные тесты, используя флаг **--testNamePattern** или методы **test.only** и **describe.only**. Например, если вы хотите запустить только один тест с названием should fetch user with id 1, выполните в терминале:

```bash
npm test -- --testNamePattern="should fetch user with id 1"
```

Эта команда запустит только тест с указанным названием. Флаг **--testNamePattern** принимает регулярное выражение, так вы можете запускать группу тестов, например, все тесты для **fetchUser**, указав `--testNamePattern="fetchUser"`. Это полезно, когда вы работаете над конкретной функцией и не хотите ждать, пока пройдут все тесты проекта.

*Альтернативный способ* — использовать **test.only** или **describe.only** в коде. Если добавить .only к тесту или блоку describe, Jest выполнит только этот тест или группу тестов. Например:

```javascript
describe('fetchUser function', () => {
  test.only('should fetch user with id 1', async () => {
    const id = 1;
    const result = await fetchUser(id);
    expect(result).toHaveProperty('name', 'Alice');
  });

  test('should throw error for unknown user', async () => {
    const id = 2;
    await expect(fetchUser(id)).rejects.toThrow('User not found');
  });
});
```

В этом примере Jest выполнит только тест should fetch user with id 1. Это удобно для отладки, но важно убрать .only перед отправкой кода в репозиторий, чтобы не пропустить другие тесты. Документация Jest рекомендует использовать **.only** только временно, чтобы избежать случайного исключения важных тестов.

*Иногда нужно временно исключить тест*, не удаляя его код. Для этого используйте **test.skip** или **describe.skip**. Это позволяет пропустить тест или группу тестов без их удаления. **Например**:

```javascript
describe('fetchUser function', () => {
  test('should fetch user with id 1', async () => {
    const id = 1;
    const result = await fetchUser(id);
    expect(result).toHaveProperty('name', 'Alice');
  });

  test.skip('should throw error for unknown user', async () => {
    const id = 2;
    await expect(fetchUser(id)).rejects.toThrow('User not found');
  });
});
```

Здесь тест should throw error for unknown user будет пропущен, но его код останется в файле. Это полезно, если тест временно не работает, например, из-за изменений в API, но вы планируете вернуть его позже. Согласно документации Jest, skip помогает поддерживать код тестов, не теряя прогресс.

Еще одна полезная техника — использование флага `--watch` для запуска Jest в интерактивном режиме. Выполните:

```bash
npm test -- --watch
```

В этом режиме *Jest отслеживает изменения в файлах и автоматически перезапускает тесты*. Вы также можете выбрать конкретные тесты через интерактивное меню, нажав **t** и введя шаблон названия теста. Это удобно, когда вы активно работаете над кодом и хотите быстро проверять изменения.

Для больших проектов полезно использовать флаг `--coverage`, чтобы проверить, какие части кода покрыты тестами:

```javascript
npm test -- --coverage
```

Jest сгенерирует отчет о покрытии, показывая, какие строки кода протестированы, а какие нет. Это помогает найти непокрытые участки кода и улучшить тесты.
